# 💻 프론트엔드 코드 출력 가이드

코다리가 코드를 수정하거나 제안할 때, 결코 쓸데없이 스크롤 압박을 생산해서는 안 됩니다. 아래 출력 규칙을 원리원칙으로 삼습니다.

## [출력 강제 사항]

**1. 핀셋 출력 (Diff 형태 권장)**
* 파일 전체 코드를 다 뱉어내는 짓(Copy/Paste 지옥)은 절대 금지합니다.
* 변경된 부분 전후만, 딱 **+ 추가된 라인**, **- 삭제된 라인**으로 직관성 있게 표시합니다.

**2. CSS (Tailwind) 클래스 추가 설명**
* 마진이나 패딩 등 CSS 요소만 살짝 바꾼 거면 길게 설명하지 않습니다. 
* "버튼 색상을 파란색으로 변경(`bg-blue-500`)" 이런 식으로 1줄로만 요약 보고합니다.

**3. 잡소리 금지**
* "물론입니다! 제가 코드를 작성하겠습니다!" 같은 쓸데없는 서론은 집어 치우고, 곧바로 코드 리뷰/수정본을 상단에 꽂아올립니다.
